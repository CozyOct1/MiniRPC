Feature Set

One compact framework for RPC transport, routing, and resilience.

动态代理

通过 JDK Proxy 将接口方法调用转换为 RPC 请求，消费者像调用本地方法一样调用远程服务。

Netty 长连接

客户端连接池复用 Channel，服务端通过 Netty pipeline 完成高吞吐收发与编解码。

Virtual Threads

业务方法执行默认交给 Java 21 虚拟线程，降低高并发阻塞调用下的线程成本。

服务发现

内置本地注册表，抽象接口可扩展 Redis、Nacos、Zookeeper 等注册中心。

负载均衡

支持随机、轮询和一致性 Hash，适配普通调用和需要路由稳定性的场景。

容错观测

提供超时、重试、熔断与 Prometheus 指标，输出请求量、失败率、重试数和 P95 延迟。

Open Source

代码公开，依赖清晰，适合拆解 RPC 框架的关键路径。

Bring your runtime

只需要 JDK 21 与 Maven，即可编译测试并运行 Echo 示例服务。

No external registry

默认使用本地注册表，注册中心接口可以继续扩展到 Redis、Nacos 或 Zookeeper。

Observable by default

内置 Prometheus text format 指标，便于观察请求量、失败率、重试数与延迟。

Architecture

从代理调用到服务执行，每一步都可追踪。

客户端通过 JDK Proxy 生成请求，连接池复用 Netty Channel，二进制协议负责帧边界，服务端使用虚拟线程执行业务方法，并将结果按相同协议返回。

Quickstart

Clone, test, run.

mvn test
mvn -q -DskipTests package
java -jar target/minirpc-1.0.0.jar

示例服务监听 127.0.0.1:9000，指标暴露在 http://127.0.0.1:9100/metrics。
